## なぜ作ったのか

URLにクエリパラメータを手で組み立てているとき、`&` や `=`、日本語、スペースなどをどうエンコードするか毎回迷う、という地味なストレスがありました。`encodeURIComponent` をブラウザのコンソールで叩くのも手ですが、「サッと貼り付けて結果が見たいだけ」のときには大げさです。

APIを叩く前のデバッグ、リダイレクトURLの組み立て、Webhookのパラメータ確認――こういう場面で「コピペして変換ボタンを押すだけ」のツールが手元にあると地味に効きます。自分が欲しかったので作りました。

## 主な機能

- **エンコード / デコードの双方向変換** — テキストを入れてボタンを押すだけ
- **`encodeURIComponent` と `encodeURI` の切り替え** — パラメータ全体を変換したいのか、URL構造（`/` や `:`）を保持したいのかを選べる
- **ワンクリックコピー** — 変換結果をそのままクリップボードへ
- **リアルタイム変換** — 入力に応じて即座に結果を反映

地味ですが、`encodeURIComponent` と `encodeURI` の違いを意識せず使うとバグるので、明示的に選べるようにしたのは個人的なこだわりポイントです。

## 技術的なポイント

実装はVanilla JS。フレームワークもビルドステップもありません。変換のコアは標準APIそのままです。

```js
// encodeURIComponent: 予約文字も含めて全部エンコード
encodeURIComponent("a=b&c=日本語");
// => "a%3Db%26c%3D%E6%97%A5%E6%9C%AC%E8%AA%9E"

// encodeURI: URL構造に使う文字(: / ? & =)は保持
encodeURI("https://example.com/パス?q=値");
// => "https://example.com/%E3%83%91%E3%82%B9?q=%E5%80%A4"
```

ホスティングはCloudflare Pages。静的ファイルだけなので、エッジ配信で速く、コストもかかりません。データはすべてブラウザ内で処理され、サーバーには一切送信されないので、機密を含むURLを貼っても安心して使えます。

## テスト結果

代表的なケースを手元で確認しました。

- 日本語を含むクエリ値 → 正しくパーセントエンコードされる
- すでにエンコード済みの文字列をデコード → 元に戻る（ラウンドトリップ）
- スペース → `%20` になる（`+` ではない、`encodeURIComponent`準拠）
- 絵文字を含むURL → サロゲートペアを壊さず変換できる

ラウンドトリップ（エンコード→デコードで元に戻る）が成立することを基準にしているので、安心して貼り付けられます。
